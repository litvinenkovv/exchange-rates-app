Description of the Test Case:

The task is to develop a simple currency converter that makes use of the Fixer.io API to provide current conversion rates. The design of the application should be based on the provided example image.

Test Case Requirements:

  1. Retrieve Data from the API: Your application should make a GET request to the Fixer.io API to retrieve the latest conversion rates.
  2. Currency Conversion: The application should allow users to select their base currency and a currency to convert to. It should display the corresponding conversion rate and calculated conversion value as the user enters a value.
  3. Error Handling: The application should handle potential errors gracefully. For instance, it should display an appropriate message if the API is not available or if the user enters invalid input.
  4. User Interface: The UI should be intuitive and user-friendly, you can use the below example as a point of reference and minimal requirement, however if you have some ideas about how to improve the UI/UX then by all means do so.

Submission:

Please provide the source code for the application, including all necessary files to run the application locally, In addition, please provide a brief README describing the application, how to run it, and any other important information about your implementation.

This is a real-world task that will test your ability to develop a functional, user-friendly application with real-time data retrieval. We look forward to seeing your creativity, problem-solving skills, and technical expertise in action.
